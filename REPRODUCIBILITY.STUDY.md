# General notes regarding reproducibility study: ECIR'24 Sparse Cross-Encoder

This markdown outlines notes regarding contributions, environment configurations, patches, and data dumps. The reproducibility study is explained in detail in [this paper](DSAIT-2526-4050IR-Final_paper.pdf). We've only reproduced one model, which you can find on [Hugging Face](https://huggingface.co/YoanPopov/reproduced-ecir24-sparse-512)

## 0. Additions to the Original Repository

The following files were added as part of this reproducibility study. Given our final setup, none of the original repository code was modified beyond the truncation bugfix described in Section 4.2. However, additional issues for different setups were also identified (see Section 5)

### Scripts

| File | Purpose |
|------|---------|
| `evaluate_passage.py` | End-to-end evaluation pipeline. Generates BM25 first-stage baselines via Pyserini, re-ranks them with a given cross-encoder model, and computes per-query and averaged nDCG@10. Supports both TREC DL and BEIR datasets through the `DATASETS` list. |
| `tost_equivalence.py` | Paired TOST (Two One-Sided Tests) equivalence testing. Compares each model against a baseline (full attention) to determine whether nDCG@10 differences fall within a ±epsilon margin. Applies Bonferroni correction across comparisons and includes a micro-averaged TREC DL section. |
| `layerwise_analysis.py` | Layer-wise prediction analysis. Runs inference with `output_hidden_states=True`, applies the final classifier head to each intermediate layer's CLS output, and computes nDCG@10 and Kendall's tau (rank correlation with the final layer) per layer. |
| `layerwise_plots.py` | Visualization script for the layer-wise results. Reads `layerwise.csv` and generates: per-dataset nDCG@10 line plots, averaged nDCG@10 curves (all/in-domain/out-of-domain) with standard deviation bands, per-dataset Kendall tau heatmaps, and averaged bubble heatmaps showing layer alignment between models. |

### Data Files

| File | Purpose |
|------|---------|
| `layerwise.csv` | Layer-wise analysis results for 3 models (reproduced sparse, author sparse, full attention) across 7 datasets (4 TREC DL + 3 BEIR). Columns: `Model Name`, `Dataset Name`, `Layer`, `nDCG@10`, `Kendall τ`. |

### Figures

The `reproduction-images/` directory contains all plots included in the report, generated by `layerwise_plots.py`:

- **Per-dataset nDCG line plots** (e.g., `beir-nq_ndcg_layerwise.png`) — layer-wise nDCG@10 for all 3 models on a single dataset
- **Averaged nDCG curves** (e.g., `average_ndcg_layerwise_in-domain_with_deviation.png`) — mean nDCG@10 per layer across dataset groups, with ±1 std deviation bands
- **Per-dataset Kendall tau heatmaps** (e.g., `beir-nq_heatmap_tau.png`) — cross-layer alignment between reproduced sparse and full attention models
- **Averaged bubble heatmaps** (e.g., `average_bubble_heatmap_out-domain_alignment.png`) — bubble color encodes mean Kendall tau difference, bubble size encodes consistency (inverse std deviation)

### 4.2 Truncation issue

An off-by-one error occurs during sequence truncation, resulting in an IndexError within the positional 
embedding layer. When assembling the query, document, and necessary special 
tokens (like [CLS] and [SEP]), the total token count can reach the absolute 
maximum sequence limit (e.g., exactly 512). Because positional embedding 
tables utilize zero-based indexing (e.g., 0 to 511), attempting to process 
this sequence causes the model to query an out-of-bounds index (512).

A -1 fix prevents this:
max_doc_length = self.max_length - max_query_length - 1

Note that we've already applied this fix

## 5. Possible issues 

### 5.1 Deprecated arguments
Due to updates in PyTorch/Lightning, the `verbose` argument in the learning rate scheduler is deprecated. Apply the following patch to fix the `__init__` method:

```bash
sed -i 's/super().__init__(optimizer, lr_lambda, last_epoch, verbose)/super().__init__(optimizer, lr_lambda, last_epoch)/' \
  /workspace/ecir24-sparse-cross-encoder/sparse_cross_encoder/model/warmup_schedulers.py
```

### 5.2 Prediction / Inference

Newer versions of PyTorch Lightning require full class paths in configuration files. Because `PredictionWriter` isn't imported directly in `main.py`, the short name will fail.

Patch the `predict.yaml` file to use the absolute class path:
```bash
sed -i 's|class_path: PredictionWriter|class_path: sparse_cross_encoder.model.callbacks.PredictionWriter|' \
  /workspace/ecir24-sparse-cross-encoder/sparse_cross_encoder/configs/cli/predict.yaml
```
